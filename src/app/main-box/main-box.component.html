Scenario 1 — providedIn: null

Each app-box lists CounterService in its own providers array. Angular creates a separate instance for every box — incrementing one does not affect the others.

Scenario 2 — providedIn: 'root' (singleton)

If you change CounterService to providedIn: 'root' and remove the providers array from BoxComponent, all boxes share one single instance — they all show the same count.

Property providedIn: null providedIn: 'root'
Instances created One per component One for the entire app
Shared state No — isolated Yes — shared everywhere
Requires providers: [] Yes (in component) No
Use case Per-component state App-wide state / singleton
